用户权限模块

全部使用状态码进行返回(此处状态码为http body内的，如不是http头部的状态码)
###登录功能
200 # 登录成功
201 # 登出成功
<!-->
250 # 登录失败，密码错误
251 # 登录失败，查无此用户
-->
登录失败：
状态为为250，状态为fail，错误信息在message中展示

###用户相关操作
<!-->
300 # 添加用户成功
350 # 添加用户失败，用户重名
301 # 修改用户成功
353 # 修改用户，原密码错误
json_data # 该用户获取自己成功
355 # 修改用户密码时，原密码或者新密码为空
302 # 删除用户成功
-->
对于所有操作：
成功状态码为300，状态为success
失败状态码为350，状态为fail，错误信息在message中展示

###组操作
<!-->
900 # 添加组成功
950 # 添加组失败， 用户组重名
901 # 删除组成功
json_data # 获取组信息成功
952 # 获取组信息失败，没此用户组
902 # 修改组成功
954 # 修改组失败，没有此用户组
-->
对于所有操作：
成功状态码为900，状态为success
失败状态码为950，状态为fail，错误信息在message中展示


###通用错误
354 # 加密函数Crypto报错
<!-->
360 # 所有涉及Mongo插入但是插入失败的错误
361 # 所有涉及查询Mongo但是查询失败的错误
362 # 所有涉及删除mongo记录但是删除失败的错误
-->
358 # 微信返回二维码错误
涉及到Mongodb的错误
状态码为360，状态为fail，错误信息在message中展示

####对于需要登录才能使用的工具，如果检测到默认状态没有登录，
500 #未登录

####对于需要权限才能使用的功能，如果检测到没有权限会返回
501 #没用此功能权限
